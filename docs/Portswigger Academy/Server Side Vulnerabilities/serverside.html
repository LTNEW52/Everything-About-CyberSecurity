Executive Summary

The labs reproduced fifteen issues across access control, authentication, input handling, and backend integrations. Critical findings (SQL Injection, OS Command Injection, unrestricted File Upload) enable full compromise of the app and host. High-severity findings (SSRF, weak access control, brute-force/2FA gaps) accelerate lateral movement and data exposure.

Methodology

  • Reconnaissance — read app behavior, source hints, and paths (e.g., robots.txt).
  • Scanning — map inputs/parameters through Burp Proxy; fuzz with Intruder.
  • Exploitation — validate impact using safe payloads and evidence.

Tools Used

Burp Suite (Community) Burp Intruder Burp Repeater

Findings

Each item below includes Impact and Recommendation.

SS-001 Path Traversal 7.5

Impact

Access to sensitive OS files (e.g., /etc/passwd) exposes system user data and aids credential attacks or privilege escalation.

Recommendation

  • Canonicalize and validate paths; block ../ traversal and null-byte tricks.
  • Whitelist directories and use safe file APIs.
SS-002 Insecure Admin Panel (robots.txt) 8.9

Impact

Admin endpoints disclosed via robots.txt allow direct access to privileged functions when unprotected.

Recommendation

  • Remove sensitive hints; require authentication and server-side authorization (RBAC) on all admin routes.
SS-003 Obscured URLs 6.5

Impact

Hidden endpoints embedded in client code can be enumerated and accessed directly, bypassing intended navigation controls.

Recommendation

  • Enforce authorization on the server for every sensitive endpoint.
  • Avoid exposing secret paths in client assets; rotate and monitor endpoints.
SS-005 GUID Disclosure 6.8

Impact

Public GUIDs enable identifier reuse to access another user’s resources, leading to data exposure or account impersonation.

Recommendation

  • Avoid exposing identifiers publicly; validate access to resources server-side.
  • Rotate identifiers if leaked; add authorization checks to object access.
SS-006 Vulnerable Administration 7.4

Impact

Exposed administrator credentials or predictable IDs allow unauthorized login and full backend control.

Recommendation

  • Rotate credentials; enforce MFA and IP restrictions for admin access.
  • Store salted password hashes (bcrypt/Argon2); monitor and alert on admin logins.
SS-007 Brute Force 8.8

Impact

Unrestricted guessing of usernames and passwords leads to account takeover and potential privilege escalation.

Recommendation

  • Rate-limit and add progressive delays; implement account lockout and CAPTCHA.
  • Enable MFA and monitor failed login anomalies.
SS-008 2FA Bypass 8.2

Impact

Direct navigation to protected endpoints before 2FA completion nullifies the second factor, enabling unauthorized access.

Recommendation

  • Issue a fully privileged session only after successful 2FA.
  • Re-validate 2FA for sensitive operations and prevent pre-2FA session usage.
SS-004 Access Control — Cookie Tampering 9.8

Impact

Trusting client-side flags (e.g., admin=true) enables instant privilege escalation by editing the cookie.

Recommendation

  • Perform authorization checks on the server; never rely on client-side role indicators.
  • Sign/encrypt session cookies and verify on each request.
SS-009 SSRF — Admin Access 9.0

Impact

Abusing server-side fetch to localhost/admin exposes internal admin interfaces and privileged actions.

Recommendation

  • Enforce egress allow-lists; block private ranges and localhost.
  • Validate/normalize URLs server-side; use SSRF protections and metadata service filters.
SS-010 SSRF — Internal Discovery 8.7

Impact

Enumeration of internal IPs (e.g., 192.168.0.x) reveals hidden services and broadens the attack surface for lateral movement.

Recommendation

  • Validate destinations against an allow-list; restrict DNS rebinding and redirects.
  • Segment networks; deny internal egress from web tiers by default.
SS-011 File Upload — Web Shell 10.0

Impact

Unrestricted uploads enable remote code execution via malicious web shells, leading to full host compromise.

Recommendation

  • Strict allow-list by type; verify MIME and magic bytes; AV/clam scan uploads.
  • Store outside webroot; randomize names; remove execute perms; proxy via signed URLs.
SS-012 File Upload — MIME Spoofing 8.6

Impact

Extension-only checks can be bypassed with spoofed headers, allowing dangerous files to be accepted and executed.

Recommendation

  • Verify magic bytes; enforce server-side content inspection; convert images to safe formats.
  • Rename uploads; use isolated storage and expiring access links.
SS-013 OS Command Injection 10.0

Impact

Unsanitized input passed to shell commands enables arbitrary OS execution and complete system compromise.

Recommendation

  • Use safe library APIs instead of shell; validate/escape inputs; apply allow-lists.
  • Run with least privilege; contain with AppArmor/SELinux; monitor command exec.
SS-014 SQL Injection — Data Exposure 9.8

Impact

Injection in data queries exposes sensitive records and allows tampering, impacting confidentiality and integrity.

Recommendation

  • Use parameterized queries/ORM; centralized input validation and encoding.
  • Restrict DB privileges; enable query logging and anomaly detection.
SS-015 SQL Injection — Auth Bypass 10.0

Impact

Authentication logic can be bypassed (e.g., ' OR 1=1--), granting unauthorized administrative access.

Recommendation

  • Prepared statements for all auth queries; sanitize inputs; avoid string concatenation.
  • Enforce MFA for admin users; lockouts and alerts on suspicious login patterns.
